# trifle v0.4 — Implementation Plan

This plan maps the v0.4 scoring derivation (`docs/derivation.md`) onto the trifle codebase: what changes,
where, in what order, and how it is verified. Section references like **§5** point at the derivation;
`file:symbol` points at the code.

**Source of truth.** The derivation is authoritative; the implementation bends to it. On-disk meaning changes
are *free*: bump `SCHEMA_VERSION` / the tokenizer fingerprint and the existing drift-reset path drops and
rebuilds the cache — **no migration code**. This licenses the tokenizer and storage changes below.

---

## 1. What v0.4 changes

v0.3 scores a single **integer**: IDF-weighted overlap via a 4-tier `{1,2,3,4}` df-max-relative scheme
(`crates/trifle-overlap/src/lib.rs:tier_weights`), N-free, with class-normalized rarity used only in
*selection*. There is no count credit, contamination floor, stopping rule, length null, or multi-order fusion.

v0.4 replaces the scoring layer with the derivation:

| Element | Derivation |
|---|---|
| logit-idf energy `E_g = ln((N−df_eff−κ)/(df_eff+κ))`, N-anchored | §2, §4 |
| count credit `μ = max(0, logit r)`, `r = σ` (query) / `σ(1−ε)^n` (doc) | §3, §9 |
| contamination floor `df_min = N^((ν−1)/ν)`, ceiling `E_max = (1/ν)ln N` | §4 |
| Cantelli confidence-bounded stop, comonotone per-word-**block** variance | §5 |
| saturating length null `π_g = 1−(1−p_g)^(L/L̄)` | §6 |
| bit-sliced energy planes + per-order popcount credit | §7 |
| multi-gram-order + multi-script: pool cross-script / fuse same-run, **rank-views**, class-normalized selection + per-class floor | §8 |

The storage/lifecycle layer (schema, store, postings, dict, drift, leases) is **largely untouched**.

---

## 2. Where it lands — three hot spots, two new capabilities, one plumbing change

1. **`crates/trifle-overlap`** — quantized logit-idf energy **planes** replace `tier_weights`; a NEW
   per-order non-floored **popcount** accumulator for the count credit; a **float post-pass**; the walk
   iterates the **union** (bit-sliced ∪ popcount).
2. **`src/select.rs`** — the Cantelli energy-sufficiency **stop** + per-class **floor** + skip-and-continue
   budget replace the fixed `t_max` prefix (the `df_budget == C` budget stays). Needs `E_g`, `r`, `N`, the
   `floored` flag, and per-gram `(word, script, order)` — today `select` sees only `(token, df, class)`.
3. **`src/search.rs`** — the length-null float pass (between walk and top-k); top-k ordering on the **float**
   corrected score; per-rank-view planning + RRF fusion.
4. **NEW — position/word/script-aware QUERY tokenization** in `src/tokenize.rs` (§5 block = query word; §8
   dual-order + rank-views; whitespace/delimiter-punct as window+word boundaries).
5. **NEW — plumb N (and `avgdl`, already exposed) into the scoring path.** Energy/floor/stop/null are
   N-anchored; the engine is deliberately N-free. Cleanest: compute energy in `search.rs::prepare` (which has
   N via `schema::read_seg_stats`) and feed explicit per-posting weights through
   `Counter::build_weighted_from_blobs` (the zero-copy explicit-weight path already exists).

## 4. Cross-cutting gaps to decide early (G1–G4)

- **G1 — positions/words/script on the query tokenize path.** `tokenize.rs::GramTokens::next` yields a bare
  `Gram`; offsets live only in `span()` (survivors-only, min/max); whitespace is `Common`/transparent so
  `"quick brown"` is one run with no word boundary. §5's block and §8's rank-views need a query-side
  tokenization that emits `(gram, word-id, script, order)` and makes whitespace + delimiter punctuation
  **break gram windows** (so no gram straddles two words). Keep it consistent with the index-side tokenizer
  (fingerprint!).
- **G2 — integer best-first vs float corrected score** (the most consequential refactor). The engine streams
  top-k lazily on integer buckets, but μ + the length null are floats, so §6/§7 force "score the bounded
  union (≤C), *then* sort." This reshapes `Walk`/`CounterIter`/`CandidateStream`'s lazy contract into
  accumulate-union → float post-pass → sort. O(C), but verify *flatness* (cardinality-independent op count)
  survives.
- **G3 — `seg.len` semantics.** Today total-gram-count-with-repetition (`lib.rs`), but §0/§6 `L_d` is the
  *distinct* gram count. Affects `avgdl`, the null, and the `thrash.rs` rolling-counter invariant. Decide in
  M0: redefine `seg.len` to distinct (re-index via drift-reset) — recommended, since the null is about presence.
- **G4 — `(script, order)` class key.** `welford.rs::ClassStats` is `[Welford; 256]` indexed by script byte;
  v0.4 dual-order makes a Latin trigram and a Latin bigram share a script byte but live in different df
  regimes. Re-key by `(script, order)`. `select`'s `(token, df, class)` triple gains an order field.

## 5. Multi-script & symbols (settled; derivation §8, §5)

The full resolution, cross-validated:

- **Per-gram order n (M1).** Order is per `(script, order)` class (CJK bigrams, else trigrams). A mixed-script
  query carries several orders, **pooled** into one accumulator with per-gram `r = σ(1−ε)^n`; the count credit
  is per-order `Σ_n μ_n·popcount_n`. `df_min`, `E_max`, κ, query-side σ are n-independent. `batch == serial`
  holds (per-query selection unchanged).
- **Global energy + class-normalized selection (M2).** The energy/score stays the **global** logit-idf (E1 —
  honest log-odds; class-normalizing the *energy* provably *starves* a minority script). The rarest-first
  **selection key** is the class-normalized z-score within `(script, order)` (v0.3 `welford.rs`/`select.rs`
  already do this — keep it). The budget cutoff is **skip-and-continue** (the z-score order is not
  df-monotone). A **per-class floor** seats the rarest `df>0` gram (floored or not, `df ≤ C`) of every present
  `(script, order)` class *unconditionally* → per-script representation is a true invariant.
- **Pool vs fuse + rank-views (M3).** Pool cross-script grams (disjoint, no containment); RRF-**fuse**
  same-run multi-order (sub-gram containment → otherwise triple-counts). Uniform rule: each script = **primary
  + richness-gated one-shorter secondary** (Latin 3+2, CJK 2+1). Mixed queries use **rank-views**: a
  primary-view pools every script's primary order, a secondary-view pools every script's secondary; one RRF
  over `[primary, secondary]`, `missing="omit"`. The CJK unigram secondary is a morpheme view (no candidate
  explosion — a common Han char is killed by both the energy clamp and the budget). `ΔH` is per-`(script,order)`.
- **Word segmentation (M4).** Whitespace + delimiter punctuation become gram-windowing + block boundaries
  (REQUIRED — a straddling gram breaks the one-word-one-Bernoulli assumption the §5 variance rests on).
  Intra-word punctuation (apostrophe/hyphen) is word-internal, optionally stripped. A CJK run with no spaces
  is one block (recall-safe; a length cap / lightweight segmenter is a precision refinement).
- **Symbols/numbers (M5).** `Common`-class, transparent within a run; the floor stays **on** (recall-safe; do
  not exempt — IDs are mistyped too). A per-field **doc-side opt-in** handles known structured-ID fields (no
  floor, full idf, credit). One `Common` bucket for v0.4 (general-category sub-classing = future); a per-class
  `df_min` is a future precision refinement (the global floor over-floors CJK's large bigram space — recall-safe).

| Multi-script element | Lands in | Change |
|---|---|---|
| dual-order tokenization | `tokenize.rs::DefaultTokenizer`/`WindowPolicy`/`GramTokens` | emit BOTH primary+secondary order per script; whitespace/delimiter-punct break gram windows + mark word-ids; tag each gram `(script, order, word-id)`; intra-word-punct strip option (new `Normalization`) |
| `(script,order)` class | `welford.rs::ClassStats` | re-key by `(script,order)`; add secondary-order classes |
| class-normalized selection + per-class floor + skip-continue | `src/select.rs` | KEEP z-score sort; ADD per-`(script,order)` floor seats (rarest `df>0`, `df≤C`); `df_budget` `break`→`continue`; ADD the Cantelli stop |
| dual-order postings/storage | `postings.rs`, `dict.rs`, `schema.rs` | store both orders' postings (per-`(script,order)` term partition or an order tag on `Term`); `SCHEMA_VERSION` bump |
| rank-views + per-order credit + RRF | `src/search.rs` | build primary/secondary rank-views; per-order μ popcount; RRF over rank-views; `ΔH` per-`(script,order)` |

## 6. Milestones (ordered; each ends green: `cargo test --workspace`, `clippy -D warnings`, `doc -D warnings`)

- **M0 — foundations & scaffolding.** Decide `seg.len` = distinct-gram count (G3); thread N into the scoring
  path; add `ν, κ, Δ, σ, ε, k, c, C` to `SearchOpts`/index config; `SCHEMA_VERSION` bump + drift-reset test.
- **M1 — global logit-idf energy planes.** Replace `tier_weights` with quantized logit-idf computed in
  `prepare` (has N), fed via `build_weighted*`; engine plane count `⌈log2(E_max/Δ)⌉`; guard `Δ < 2·E_floored`.
  *Accept:* energy weights match the derivation on fixtures; existing typo/unicode/basic tests still pass.
- **M2 — count credit μ + contamination floor + floored flag.** Per-order non-floored popcount; float ranking
  key; query-side floor; `μ = max(0, logit r)` with the §9 concentration cap applied **before** the credit
  feeds both the score and the null. *Accept:* junk-inertness + floor↔μ ordering (the `df≈9000` crossover);
  the cap demotes commons-heavy candidates.
- **M3 — length null + float post-pass + top-k-after-floats (the G2 reshape).** Per-candidate null in
  `search.rs`; restructure the walk/stream to accumulate the bounded union, then post-pass + sort. *Accept:*
  length-bias regression test; the union recovers count-only + floored-only candidates (`Δ < 2·E_floored`).
- **M4 — Cantelli stop + per-class floor + position/word/script query tokenization (G1).** New query
  tokenization emits `(gram, word-id, script, order)`; comonotone-block stop + class-normalized z-score key +
  per-class floor + skip-and-continue in `select.rs`. *Accept:* the stop fires recall-safely (`r ≥ c²/(B+c²)`);
  the C1 starved-query case returns the bigram set, not empty; multi-script queries seat every present class;
  `thrash.rs` oracle survives.
- **M5 — dual-order tokenization + storage + rank-view RRF fusion (§8).** `DefaultTokenizer` emits both
  orders for trigram-primary scripts + the CJK unigram secondary; postings/dict keyed by `(script,order)`;
  per-`(script,order)` `ΔH`; `needs_secondary` gate; rank-view RRF. *Accept:* fusion beats pooling on the
  double-count fixture; CJK gets its bigram+unigram views; a mixed-script query pools the primary view and
  fuses the rank-views.
- **Cross-cutting:** `(script,order)` welford key (G4); repurpose `stats()`/`WeightStepHint` → Δ + stop
  telemetry; update `tests/` + `tests/common`.

## 7. Requirements & invariants to preserve

From `CLAUDE.md` + the derivation:

- **`batch == serial`** — per-query selection only; now also the per-query stop (a pure function of this
  query's tokens + the shared df/zstats snapshot).
- **No ghost rows**; **dictionary-generation guard** (a read racing `rebuild` → `Error::Busy`); **monotonic
  ids + atomic shadow swap** for `rebuild`; **drift-reset** (drop, never migrate); **no sleeps** (surface
  `Error::Busy`); **flatness** (engine op count cardinality-independent — preserve through the energy planes);
  **one tokenizer on index and query** (now: both orders on both sides).
- **New recall-safety properties the derivation earns** — the stop's mean-honest + variance-conservative
  posture (§5/§11), the union recovering count-only/floored-only candidates, top-k strictly after the float
  post-pass, the per-class floor's per-script representation invariant.

## 8. Testing strategy

- Extend `tests/typo.rs` (stop recall-safety; the C1 starved→bigram case; burst/OCR recall),
  `tests/unicode.rs` (mixed-script pooling, per-`(script,order)` class, CJK bigram+unigram, symbols/numbers,
  the per-class floor seating every present script), `tests/basic.rs` (energy ordering, floor↔μ, length null,
  concentration cap), `tests/stream.rs` (union/post-pass/top-k-after-floats), `tests/drift.rs` (`SCHEMA_VERSION`
  bump, `seg.len` redefinition).
- **`tests/thrash.rs`** proptest oracle: update the reference model to the v0.4 scoring (or assert the
  recall/provenance invariants that survive the scoring change); the rolling-counter reconciliation must track
  the `seg.len` redefinition (G3).
- Numerical fixtures from the review scratch scripts (variance/domination/null/plane-count checks under
  `$CLAUDE_JOB_DIR/tmp`) → unit tests for the energy, the comonotone-block variance, null separability, and
  the plane-count bound.

## 9. Risks & sequencing notes

- **G2** (integer-walk → float-post-pass) is the single most consequential refactor; land M3 carefully and
  re-verify flatness.
- **G3** (`seg.len` distinct vs total) touches `avgdl` + the null + a `thrash` invariant — decide in M0.
- **§8 fusion** is the largest surface; sequence it last (M5) and keep it behind the `needs_secondary`
  richness gate so single-order queries pay nothing.
- The stop needs **word/script metadata the tokenizer doesn't yet produce** (G1/M4) — a new query-path
  tokenization entry point kept consistent (fingerprint) with the index side.
- Deferred (recall-safe, noted in §11): a **per-class `df_min`** (the global floor over-floors large-vocabulary
  scripts), general-category **sub-classing** of `Common`, the **cross-word** stop-variance residual, and a
  CJK-block **length cap**. None block v0.4.
