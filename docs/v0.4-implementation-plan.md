# trifle v0.4 — Implementation Plan

This plan maps the v0.4 scoring derivation (`docs/derivation.md`) onto the trifle codebase: what changes,
where, in what order, and how it is verified. Section references like §5 point at the derivation; `file:symbol`
points at the code.

Source of truth: the derivation is authoritative and the implementation follows it. On-disk meaning changes
are cheap — bump `SCHEMA_VERSION` or the tokenizer fingerprint and the existing drift-reset path drops and
rebuilds the cache, so there is no migration code. That is what licenses the tokenizer and storage changes
below.

---

## 1. What v0.4 changes

v0.3 scores one integer: IDF-weighted overlap via a 4-tier `{1,2,3,4}` df-max-relative scheme
(`crates/trifle-overlap/src/lib.rs:tier_weights`), N-free, with class-normalized rarity used only in selection.
There is no count credit, contamination floor, stopping rule, length null, or multi-order fusion.

v0.4 replaces the scoring layer with the derivation:

| Element | Derivation |
|---|---|
| logit-idf energy `E_g = ln((N−df_eff−κ)/(df_eff+κ))`, N-anchored | §2, §4 |
| count credit `μ = max(0, logit r)`, `r = σ` (query) / `σ(1−ε)^n` (doc) | §3, §9 |
| contamination floor `df_min = N^((ν−1)/ν)`, ceiling `E_max = (1/ν)ln N` | §4 |
| Cantelli confidence-bounded stop, comonotone per-word-block variance | §5 |
| saturating length null `π_g = 1−(1−p_g)^(L/L̄)` | §6 |
| bit-sliced energy planes + per-order popcount credit | §7 |
| multi-gram-order and multi-script: pool cross-script / fuse same-run, rank-views, class-normalized selection + per-class floor | §8 |

The storage and lifecycle layer (schema, store, postings, dict, drift, leases) is largely untouched.

---

## 2. Where it lands — three hot spots, two new capabilities, one plumbing change

1. `crates/trifle-overlap` — quantized logit-idf energy planes replace `tier_weights`; a new per-order
   non-floored popcount accumulator for the count credit; a float post-pass; the walk iterates the union of the
   bit-sliced and popcount candidates.
2. `src/select.rs` — the Cantelli energy-sufficiency stop, the per-class floor, and the skip-and-continue
   budget replace the fixed `t_max` prefix (the `df_budget == C` budget stays). It needs `E_g`, `r`, `N`, the
   `floored` flag, and per-gram `(word, script, order)`; today `select` sees only `(token, df, class)`.
3. `src/search.rs` — the length-null float pass (between the walk and top-k); top-k ordering on the float
   corrected score; per-rank-view planning and RRF fusion.
4. New: position/word/script-aware query tokenization in `src/tokenize.rs` (the §5 block is a query word; §8
   needs dual-order and rank-views; whitespace and delimiter punctuation become window and word boundaries).
5. New: plumb N (and `avgdl`, already exposed) into the scoring path. Energy, floor, stop, and null are
   N-anchored, while the engine is N-free by design. The cleanest route is to compute energy in
   `search.rs::prepare` (which has N via `schema::read_seg_stats`) and feed explicit per-posting weights through
   `Counter::build_weighted_from_blobs` (the zero-copy explicit-weight path already exists).

## 3. Design → code mapping

| Derivation element | Lands in (`file:symbol`) | Change | Notes |
|---|---|---|---|
| logit-idf energy `E_g` (§2,§4) | `trifle-overlap::tier_weights` → new `energy_weights`; `search.rs::prepare` | compute `E_g`, quantize `wq = max(0, round(E_g/Δ))`; feed via `build_weighted*` | N must reach scoring (the engine is N-free). Guard `Δ < 2·E_floored` (§7). |
| count credit `μ` (§3,§9) | new post-pass in `trifle-overlap` + `search.rs` | `μ·popcount(non-floored matched)`, bucketed by order (`Σ_n μ_n·popcount_n`); the ranking key becomes a float | engine `Scored.overlap` counts all matched grams, so a non-floored, per-order popcount is needed. μ is the capped value (concentration cap, §9), and the null uses the same capped μ. |
| contamination floor (§4) | energy fn + `select.rs` floored flag | `df_eff = max(df, df_min)`, query-side only; gates μ and the stop's exclusion | a per-gram `floored` bool, set only on the query-side channel, threaded selection→energy→credit. |
| Cantelli stop + comonotone-block variance (§5) | `src/select.rs` (replaces the `t_max`/`df_budget` cutoff) | `Σ r·max(0,E) − c·σ ≥ ln(N/k)`, `σ² = r(1−r)Σ_blocks(Σe)²`; per-class floor; budget `break`→`continue` | needs per-gram `word` (block id), `r`, N, and the floored flag; word boundaries do not exist on the tokenize path today. |
| saturating length null (§6) | new per-candidate float pass in `search.rs` (post-walk, pre-top-k) | subtract `Σ π_g(L)·weight(g)`; `K_rare` precomputed, commons per-candidate | `avgdl` (L̄) and N are already on `CandidateStream` but unused. `seg.len` is total-with-repetition; the §0 `L_d` is distinct — redefine to distinct (re-index via drift) or store both. |
| bit-sliced energy + popcount (§7) | `trifle-overlap::Counter`/`Walk` | planes hold quantized energy; add the per-order non-floored popcount; iterate the union | integer best-first vs float corrected score: §6/§7 require top-k after the float post-pass over the bounded union, which abandons the lazy best-first contract — accumulate the union, post-pass, then sort. |
| multi-order + multi-script (§8) | `tokenize.rs` + `postings`/`dict` + `search.rs` | rank-views, dual-order index, per-(script,order) `ΔH`, the per-script `starved` gate, RRF | the largest change. See §5 below. |
| per-script-class handling (§8,§10) | `welford.rs` (keep) + per-order params | the class key grows from `script` to `(script, order)`; per-gram n drives r, μ, and the floor window | class-normalized rarity stays on the selection side; the energy is global. |

## 4. Cross-cutting gaps to decide early (G1–G4)

- **G1 — positions, words, and script on the query tokenize path.** `tokenize.rs::GramTokens::next` yields a
  bare `Gram`; offsets live only in `span()` (survivors only, min/max); whitespace is `Common`/transparent, so
  `"quick brown"` is one run with no word boundary. The §5 block and the §8 rank-views need a query-side
  tokenization that emits `(gram, word-id, script, order)` and makes whitespace and delimiter punctuation break
  gram windows, so no gram straddles two words. It must stay consistent with the index-side tokenizer (the
  fingerprint).
- **G2 — integer best-first vs float corrected score.** This is the most involved refactor. The engine streams
  top-k lazily on integer buckets, but μ and the length null are floats, so §6/§7 require scoring the bounded
  union and then sorting. This reshapes the lazy contract of `Walk`/`CounterIter`/`CandidateStream` into
  accumulate-union, float post-pass, sort. It stays O(C), but flatness (a cardinality-independent op count) must
  be checked through the change.
- **G3 — `seg.len` semantics.** Today it is the total gram count with repetition (`lib.rs`), but the §0/§6
  `L_d` is the distinct gram count. This affects `avgdl`, the null, and the `thrash.rs` rolling-counter
  invariant. Decide in M0: redefine `seg.len` to distinct (re-index via drift-reset), which is recommended
  since the null is about presence.
- **G4 — `(script, order)` class key.** `welford.rs::ClassStats` is `[Welford; 256]` indexed by the script
  byte; v0.4's dual-order tokenization makes a Latin trigram and a Latin bigram share a script byte while
  living in different df regimes. Re-key by `(script, order)`; the `(token, df, class)` triple in `select`
  gains an order field.

## 5. Multi-script and symbols (derivation §8, §5)

- **Per-gram order n (M1).** Order is a property of the `(script, order)` class (CJK bigrams, else trigrams). A
  mixed-script query carries several orders, pooled into one accumulator with per-gram `r = σ(1−ε)^n`; the count
  credit is per-order, `Σ_n μ_n·popcount_n`. `df_min`, `E_max`, κ, and query-side σ are n-independent.
  `batch == serial` still holds, since per-query selection is unchanged.
- **Global energy + class-normalized selection (M2).** The energy and score stay the global logit-idf;
  class-normalizing the energy only lowers a minority gram's weight (within-class df-fraction ≥ global), so it
  starves a minority script. The rarest-first selection key is the class-normalized z-score within
  `(script, order)`, which v0.3 `welford.rs`/`select.rs` already compute — keep it. The budget cutoff is
  skip-and-continue, because the z-score order is not df-monotone. A per-class floor seats the rarest df>0 gram
  (floored or not, df ≤ C) of every present `(script, order)` class unconditionally, so per-script
  representation is an invariant for every present class with an in-budget gram.
- **Pool vs fuse, and rank-views (M3).** Pool cross-script grams (disjoint, no containment); RRF-fuse same-run
  multi-order grams (sub-gram containment would otherwise triple-count). The uniform rule: each script runs its
  primary order plus a richness-gated one-shorter secondary (Latin 3+2, CJK 2+1). A mixed query uses rank-views
  — a primary view pools every script's primary order, a secondary view pools every script's secondary; one RRF
  over `[primary, secondary]` with `missing="omit"`. The CJK unigram secondary is a morpheme view and does not
  explode the candidate set (a common Han character is killed by both the energy clamp and the budget). `ΔH` is
  per-`(script, order)`.
- **Word segmentation (M4).** Whitespace and delimiter punctuation become gram-windowing and block boundaries.
  This is required: a gram straddling two words breaks the one-word-one-Bernoulli assumption the §5 variance
  rests on. Intra-word punctuation (apostrophe, hyphen) is word-internal and optionally stripped. A CJK run with
  no spaces is one block, which is recall-safe; a length cap or lightweight segmenter is a later precision
  refinement.
- **Symbols and numbers (M5).** These are `Common`-class and transparent within a run; the floor stays on
  (recall-safe, and IDs are mistyped too, so they should not be exempt). A per-field doc-side opt-in handles
  known structured-ID fields (no floor, full idf, credit). v0.4 uses one `Common` bucket; a general-category
  sub-class split and a per-class `df_min` (the global floor over-floors CJK's large bigram space) are later
  precision refinements.

| Multi-script element | Lands in | Change |
|---|---|---|
| dual-order tokenization | `tokenize.rs::DefaultTokenizer`/`WindowPolicy`/`GramTokens` | emit both the primary and secondary order per script; whitespace and delimiter punctuation break gram windows and mark word-ids; tag each gram `(script, order, word-id)`; an intra-word-punctuation strip option (new `Normalization`) |
| `(script, order)` class | `welford.rs::ClassStats` | re-key by `(script, order)`; add the secondary-order classes |
| class-normalized selection + per-class floor + skip-continue | `src/select.rs` | keep the z-score sort; add per-`(script, order)` floor seats (rarest df>0, df ≤ C); change the `df_budget` cutoff from `break` to `continue`; add the Cantelli stop |
| dual-order postings/storage | `postings.rs`, `dict.rs`, `schema.rs` | store both orders' postings (a per-`(script, order)` term partition, or an order tag on `Term`); bump `SCHEMA_VERSION` |
| rank-views + per-order credit + RRF | `src/search.rs` | build the primary and secondary rank-views; per-order μ popcount; RRF over the rank-views; `ΔH` per-`(script, order)` |

## 6. Milestones (ordered; each ends green: `cargo test --workspace`, `clippy -D warnings`, `doc -D warnings`)

- **M0 — foundations and scaffolding.** Decide `seg.len` = distinct-gram count (G3); thread N into the scoring
  path; add `ν, κ, Δ, σ, ε, k, c, C` to `SearchOpts`/index config; bump `SCHEMA_VERSION` and add a drift-reset
  test.
- **M1 — global logit-idf energy planes.** Replace `tier_weights` with quantized logit-idf computed in
  `prepare` (which has N), fed via `build_weighted*`; `⌈log2(E_max/Δ + 1)⌉` is the *single-gram* weight
  bit-width bound (the engine sizes the accumulator's planes to `bits(Σ wq)`, not this figure); guard
  `Δ < 2·E_floored`. Note the contamination floor is **split**: the energy-side `df_eff = max(df, df_min)` cap
  lands in M1 (it is intrinsic to logit-idf and required by the `Δ < 2·E_floored` guard), while the
  floored-**flag** → credit/stop machinery lands in M2. Accept: energy weights match the derivation on fixtures,
  and the existing typo/unicode/basic tests still pass.
- **M2 — count credit μ, contamination floor (flag), floored flag.** The per-gram `floored` **flag** threaded
  selection → credit → stop (the energy-side `df_eff` cap already landed in M1); per-order non-floored popcount;
  float ranking key; the floor query-side only; `μ = max(0, logit r)` with the §9 concentration cap applied
  before the credit feeds both the score and the null. Accept: junk-inertness and the floor↔μ ordering (the
  df≈9000 crossover); the cap demotes commons-heavy candidates.
- **M3 — length null, float post-pass, top-k after the floats (the G2 reshape).** The per-candidate null in
  `search.rs`; restructure the walk/stream to accumulate the bounded union, then post-pass and sort. Accept: a
  length-bias regression test; the union recovers count-only and floored-only candidates (`Δ < 2·E_floored`).
- **M4 — Cantelli stop, per-class floor, and position/word/script query tokenization (G1).** New query
  tokenization emits `(gram, word-id, script, order)`; the comonotone-block stop, the class-normalized z-score
  key, the per-class floor, and skip-and-continue in `select.rs`. Accept: the stop fires recall-safely
  (`r ≥ c²/(B+c²)`); a too-short single-script query reaches its secondary view rather than returning empty (the
  per-script `starved` gate subsumes "produced no primary gram"); multi-script queries seat every present class;
  the `thrash.rs` oracle survives.
- **M5 — dual-order tokenization, storage, and rank-view RRF fusion (§8).** `DefaultTokenizer` emits both
  orders for trigram-primary scripts plus the CJK unigram secondary; postings/dict keyed by `(script, order)`;
  per-`(script, order)` `ΔH`; the per-script `starved` gate; rank-view RRF. Accept: fusion beats pooling on the
  double-count fixture; CJK gets its bigram and unigram views; a mixed-script query pools the primary view and
  fuses the rank-views.
- **Cross-cutting:** the `(script, order)` welford key (G4); repurpose `stats()`/`WeightStepHint` toward Δ and
  stop telemetry; update `tests/` and `tests/common`.

## 7. Requirements and invariants to preserve

From `CLAUDE.md` and the derivation:

- `batch == serial`: per-query selection only, and now the per-query stop too — a pure function of this query's
  tokens and the shared df/zstats snapshot.
- No ghost rows; the dictionary-generation guard (a read racing `rebuild` surfaces `Error::Busy`); monotonic
  ids and the atomic shadow swap for `rebuild`; drift-reset (drop, never migrate); no sleeps (surface
  `Error::Busy`); flatness (the engine op count stays cardinality-independent, so preserve it through the energy
  planes); one tokenizer on index and query (now both orders on both sides).
- The recall-safety properties the derivation establishes: the stop's posture of not over-stating the mean and
  not under-stating the variance (§5/§11); the union recovering count-only and floored-only candidates; top-k
  applied strictly after the float post-pass; the per-class floor's per-script representation invariant.

## 8. Testing strategy

- Extend `tests/typo.rs` (stop recall-safety; the short-query-reaches-secondary case; burst/OCR recall),
  `tests/unicode.rs` (mixed-script pooling, the per-`(script, order)` class, CJK bigram and unigram, symbols
  and numbers, the per-class floor seating every present script), `tests/basic.rs` (energy ordering, floor↔μ,
  the length null, the concentration cap), `tests/stream.rs` (the union, the post-pass, top-k after the
  floats), `tests/drift.rs` (the `SCHEMA_VERSION` bump, the `seg.len` redefinition).
- `tests/thrash.rs` proptest oracle: update the reference model to the v0.4 scoring, or assert the
  recall/provenance invariants that survive the scoring change; the rolling-counter reconciliation must track
  the `seg.len` redefinition (G3).
- Numerical fixtures for the energy, the comonotone-block variance, null separability, and the plane-count
  bound become unit tests.

---

## 9. Risks and sequencing

- G2 (the integer-walk to float-post-pass change) is the most involved refactor; land M3 carefully and re-check
  flatness.
- G3 (`seg.len` distinct vs total) touches `avgdl`, the null, and a `thrash` invariant — decide it in M0.
- §8 fusion is the largest surface; sequence it last (M5) and keep it behind the per-script `starved` gate so
  single-order queries pay nothing.
- The stop needs word and script metadata the tokenizer does not yet produce (G1/M4): a new query-path
  tokenization entry point, kept consistent with the index side via the fingerprint.
- The unconditional per-class floor seats relax the strict work bound. The candidate union is bounded by
  `(1+#classes)·C` rather than `C` (§7), still O(C) for a fixed, small number of script classes.
- Deferred and recall-safe (noted in §11): a per-class `df_min` (the global floor over-floors large-vocabulary
  scripts), a general-category sub-class split of `Common`, the cross-word stop-variance residual, and a
  CJK-block length cap. None block v0.4.
